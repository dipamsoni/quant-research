---

Read `docs/CURRENT_PHASE.md` to find the active phase.

Then read the corresponding `docs/phases/phase-<N>.md` and:

1. Report which phase is active and its goal
2. List acceptance criteria, marking which are done (read CLAUDE.md or PR history if helpful)
3. Show the next 3 pending tasks from the task list
4. Identify any out-of-scope work the user might be tempted to do
5. Recommend the single next action

Be concise. The user is using this to orient themselves quickly, not get a wall of text.
